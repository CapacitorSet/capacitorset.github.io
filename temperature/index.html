Obiettivo

Si intende studiare la temperatura media stagionale per un appartamento nella città di Brescia.

Procedimento

A grandi passi, si intende seguire questo procedimento:
  1. Raccolta dei dati;
  2. Elaborazione (calcolo della media) per ogni stazione;
  3. Interpolazione spaziale;
  4. Documentazione.

Raccolta dati

I dati sono stati richiesti tramite la piattaforma Web dell'ARPA Lombardia.

Per ottenere la massima precisione si è deciso di raccogliere i dati per le due centraline di Brescia (Pastori e Ziziola) su base suboraria, per minimizzare gli errori di discretizzazione.

Errori di discretizzazione

Si definisce errore di discretizzazione quello dovuto alla conversione di una funzione continua a valori continui \[f:\mathbb R\rightarrow S\] in una discreta a valori discreti \[g:\mathbb N\rightarrow S'\] (con \(S'\) discreto).

Problema: interpolazione

Si dispone di dati limitati (i dati non sono disponibili appartamento per appartamento): è quindi necessario fare un'interpolazione nello spazio.

Considerato che si dispone solamente di 2 data source con valori molto simili (la differenza tra le medie è circa 0,9 °C), si decide di ignorare questo problema, considerando la temperatura come un campo scalare uniforme ($\nabla T=0$).

Problema: calcolo della media

Si dispone di circa 100k datapoint, e si vuole determinare un unico valore di temperatura, noto come "media stagionale".

Formalmente, date N tuple \[\langle\text{Tempo}\in\mathbb N, \text{Temperatura}\in\mathbb R, \text{Posizione}\in\mathbb R^2\rangle\] si vuole ricavare un campo scalare \[T_{media} = \mathbb R^2\rightarrow\mathbb R\]

Tipi di media

Cerchiamo una funzione che dati N rilevamenti \(x\in\mathbb R\) produca un valore medio \(x'\in\mathbb R\)\[media:\mathbb R^N\rightarrow\mathbb R\]

Tradizionalmente, si usano queste medie:

  • Aritmetica, \(\frac 1n\sum_{i=0}^nx_i\);
  • Armonica, l'inverso della media dei reciproci;
  • Geometrica, \(\sqrt[n]{\prod_{i=0}^nx_i}\).

Scelta della media

Si sa che la media geometrica attribuisce molto più peso ai valori più piccoli: questa è una caratteristica non desiderabile. Lo stesso vale per la media armonica.

Scegliamo quindi la media classica (aritmetica):\[media({x_0,x_1,\dots x_n})=\frac 1n\sum_{i=0}^nx_i\]

Commento

Il box plot mostra che la stazione di rilevamento "Pastori" tende a registrare temperature leggermente più alte, e mostra dati più "sparsi" (lo scarto interquartile è maggiore). Effettivamente, questi dati presentano $\sigma^2=4,577$, mentre quelli rilevati in via Ziziola hanno $\sigma^2=4,121$.

Software impiegato

  • Rappresentazione: MySQL
  • Elaborazione: R
  • Presentazione: Reveal.js
  • Pubblicazione: GitHub Pages

Rappresentazione: MySQL 5.5

I dati sono stati immagazzinati in un database MySQL in 3NF: una tabella mappa gli ID delle centraline ai rispettivi nomi, e un'altra raccoglie i dati nella forma "ID centralina, data, temperatura".

Con MySQL si è immagazzinato i dati, li si è filtrati (rimozione valori non validi), e si sono compiute semplici analisi. Si sono poi esportati i dati invernali in CSV.

Elaborazione: R 3.0

I dati sono stati importati in R, che ha consentito di trovare velocemente la media e i boxplot con due semplici comandi: rispettivamente mean e boxplot.

Per comodità si è usato un frontend grafico per R, Rstudio.

Presentazione: Reveal.js

Questa presentazione è stata scritta in HTML con il framework Reveal.js, che permette di creare flessibilmente presentazioni eleganti in tempi minimi. Inoltre, la rende accessibile a qualsiasi dispositivo dotato di browser: cellulari, tablet, Surface o desktop.
